Design Process

This project followed an iterative engineering design process to move from concept to validated real-time biomechanical system.

01

Define Problem

Identified a key Pilates challenge: users struggle to self-correct subtle posture errors without an instructor.

02

Set Requirements

Established design targets for high detection accuracy, low latency, stable feedback, and practical embedded implementation.

03

Prototype Hardware

Built multiple mat prototypes with varied IMU placement and force sensor layouts to compare coverage, reliability, and signal quality.

04

Develop Algorithms

Implemented low-latency C++ filtering and feature extraction to mitigate noise and produce robust motion-force indicators.

05

Validate & Iterate

Tested against real movement scenarios, measured false detections, and iteratively tuned thresholds and fusion logic.

06

Optimize Performance

Refined runtime paths and system integration to achieve response times under approximately 50ms while preserving 95% accuracy.
